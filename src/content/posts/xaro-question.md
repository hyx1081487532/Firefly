# 简要说明
使用内网穿透的IP地址不固定，而Xaero地图模组的地图数据是基于服务器IP地址来存储的,
所以会出现以前的地图数据未被启用,我们需要做的是跟重命名文件夹名称来进行重新对旧地图数据的定位,
这是一个应对内网穿透的一个常用方法, OK!让我们开始

# 🎯 操作步骤

## 第一步：定位Xaero's文件夹

1. 打开你的Minecraft游戏目录
2. 进入版本文件夹：`.minecraft/versions/你的游戏版本/`
3. 找到 `xaero` 文件夹

**完整路径示例：**

你的Minecraft根目录/.minecraft/versions/1.21.1/xaero/


## 第二步：识别相关文件夹

在 `xaero` 文件夹中，你会看到两个重要的子文件夹：

```markdown
xaero/
├── minimap/          # 小地图数据
└── world-map/        # 世界地图数据
```

## 第三步：修改文件夹名称

### 对于 minimap 文件夹：

1. 进入 `minimap` 文件夹
2. 你会看到类似这样的文件夹：
   
  - `Multiplayer_旧IP地址:端口`
  - `Multiplayer_新IP地址:端口`

3. **操作**：
   - 删除 `Multiplayer_新IP地址:端口` 文件夹（如果还没有进入新服务器，可跳过此步）
   - 将 `Multiplayer_旧IP地址:端口` 重命名为 `Multiplayer_新IP地址:端口`

### 对于 world-map 文件夹：

1. 进入 `world-map` 文件夹
2. 执行相同的重命名操作

## 第四步：验证修改

完成重命名后，文件夹结构应该如下：
```markdown
xaero/
├── minimap/
│   └── Multiplayer_新IP地址:端口/
└── world-map/
    └── Multiplayer_新IP地址:端口/
```

# 📝 实际示例

**假设服务器IP从 `123.45.67.89:25565` 更改为 `98.76.54.32:25565`：**

1. **原文件夹名称：**
   
   Multiplayer_123.45.67.89:25565


2. **新文件夹名称：**
   
   Multiplayer_98.76.54.32:25565


3. **操作：**
   - 删除（如存在）：`Multiplayer_98.76.54.32:25565`
   - 重命名：`Multiplayer_123.45.67.89:25565` → `Multiplayer_98.76.54.32:25565`

# ⚠️ 注意事项

1. **备份重要数据**：操作前建议备份整个xaero文件夹
2. **端口号重要**：确保包含完整的IP地址和端口号（如：`192.168.1.100:25565`）
3. **大小写敏感**：文件夹名称区分大小写
4. **特殊字符**：如果IP包含特殊字符，需要使用正确的转义

# 📞 故障排除

**如果地图数据丢失：**
1. 检查文件夹名称是否正确
2. 确认IP地址和端口号完全匹配
3. 检查文件权限
4. 恢复备份数据

**需要帮助？** 联系服务器管理员获取当前的准确IP地址和端口号。
